Base de Datos

Esta base de datos está diseñada para almacenar metadata extraída de la API de Twitter con filtros de busqueda establecidos. El objetivo es proveer una plataforma inicial de analisis hacia la detección de trata de personas y formas de esclavitud moderna en esta red social.

La base de datos esta compuesta por

Tweet

La información de un tweet puede contener varios campos que proporcionan detalles importantes sobre el contenido del tweet y su autor. Además de información adicional en la base de datos.

Palabras Claves

Este campo almacena datos relevantes para clasificar y filtrar tweets relacionados con la trata de personas, incluyendo métricas e indicadores para búsquedas precisas

Errores

Este campo registra los errores en la inserción de datos desde el código python, relevante para identificar y solucionar problemas técnicos y mantener la calidad de los datos almacenados.

Para el diseño de la base de datos, se han definido palabras claves relevantes para la identificación de contenido relacionado con la trata de personas en Twitter. Estas palabras claves se utilizan para filtrar y clasificar los Tweets recopilados en la base de datos.

A partir de ello, el código principal escrito en Python realiza solicitudes a la API de Twitter, filtrando el contenido a partir de textos de búsqueda pre-determinados desde el objeto 'Palabras Claves', creando así la clase 'Tweets'. La información se almacena consecutivamente en MongoDB como base de datos. El código de llamada a la base de datos en Python ingresa los objetos 'Tweet' y 'Palabras Claves' y crea el objeto 'Errores'en el cual se ingresan todos los errores ocurridos al momento de la inserción en la base de datos, conteniendo el ID del tweet, el error y una descripción completa para identificar y solucionar problemas técnicos.
