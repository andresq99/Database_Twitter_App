Base de Datos

Esta base de datos está diseñada para almacenar metadata extraída de la API de Twitter con filtros de búsqueda establecidos. El objetivo es proveer una plataforma inicial de análisis hacia la detección de trata de personas y formas de esclavitud moderna en esta red social.

La base de datos esta compuesta por

Tweet

La información de un tweet puede contener varios campos que proporcionan detalles importantes sobre el contenido del tweet y su autor. Además de información adicional en la base de datos.

Palabras Claves

Este campo almacena datos relevantes para clasificar y filtrar tweets relacionados con la trata de personas, incluyendo métricas e indicadores para búsquedas precisas

Errores

Este campo registra los errores en la inserción de datos desde el código python, relevante para identificar y solucionar problemas técnicos y mantener la calidad de los datos almacenados.

Para el diseño de la base de datos, se han definido palabras claves relevantes para la identificación de contenido relacionado con la trata de personas en Twitter. Estas palabras claves se utilizan para filtrar y clasificar los Tweets recopilados en la base de datos.

A partir de ello, el código principal escrito en Python realiza solicitudes a la API de Twitter, filtrando el contenido a partir de textos de búsqueda pre-determinados desde el objeto 'Palabras Claves', creando así la clase 'Tweets'. La información se almacena consecutivamente en MongoDB como base de datos. El código de llamada a la base de datos en Python ingresa los objetos 'Tweet' y 'Palabras Claves' y crea el objeto 'Errores'en el cual se ingresan todos los errores ocurridos al momento de la inserción en la base de datos, conteniendo el ID del tweet, el error y una descripción completa para identificar y solucionar problemas técnicos.

Este campo indica que el contenido puede reconocerse como confidencial.

"possibly_sensitive":"false"

Métricas de participación pública para el Tweet en el momento de la solicitud.

reply_settings String

Muestra quién puede responder a un Tweet determinado. Los campos devueltos son "everyone", "mentioned_users" y "followers".

"reply_settings":"everyone"

"text":"Solo en el canal de Telegram @RodrigoPoloSinCensura, dinámica de #AMA Ask Me Anything en los comentarios del audio 😉 "

Identificadores únicos que indican todas las versiones de un Tweet. Para los Tweets sin ediciones, habrá un ID. Para los Tweets con un historial de ediciones, habrá varios ID, organizados en orden ascendente que reflejan el orden de las ediciones.

author Object

El objeto author contiene metadatos de la cuenta de usuario de Twitter que describen al usuario al que se hace referencia. Este contiene: El nombre de usuario, identificador o alias de Twitter con el que este usuario se identifica. La fecha UTC y hora en que se creó la cuenta de usuario en Twitter. El texto de la descripción del perfil de este usuario. La ubicación especificada en el perfil del usuario, si el usuario proporcionó una. La dirección URL de la imagen de perfil de este usuario, tal como se muestra en el perfil del usuario. Detalles sobre la actividad de este usuario. Si este usuario es un usuario verificado de Twitter. El objeto "_twarc" contiene datos de la libreria de extracción utilizada. "hashtag" contiene la palabra clave utilizada para la busqueda del Tweet y author desde la API. Finalmente el campo "inserted_at_database" contiene informacion de fecha y hora de almacenamiento del tweet en la base de datos

Palabras Claves

Campo Tipo Descripción Ejemplo
_id ObjectId

El identificador único de la palabra clave solicitada.

"_id":"ObjectId('123123123123')"

language String

Idioma de la palabra clave.

"language":"spanish"

keyword String

Palabra clave.

"keyword": "Esclavo"

query String

Solicitud de la palabra clave.

"query":"Esclavo"

Hashtag String

Hashtag de la palabra clave para su busqueda.

"Hashtag":"#Esclavo"

english_translation String

Tradución en inglés de la palabra clave .

"english_translation":"Slave (male)"

meaning String

Significado de la palabra clave.

"meaning":"Submissive male in BDSM"

age_flag String

Indicador booleano de peligro en edad.

"age_flag":"False"

trafficking_flag String

Indicador booleano de peligro de tráfico.

"trafficking_flag":"True"

movement_flag String

Indicador booleano de peligro de desplazamiento.

"movement_flag":"False"

Errores

Campo Tipo Descripción Ejemplo
_id ObjectId

El identificador único del Error.

"_id":"ObjectId('123123123123')"

tweet_id String

El identificador único del Tweet donde ocurre el error.

"tweet_id":"1563949999859302401"

error Object

Objeto que contiene la descripción completa del error ocurrido al momento de la inserción de la información hacia la base de datos. Esta contiene: Index. Código. Mensaje del error ocurrido.

"error": {
      "index": 0,
      "code": 11000,
      "KeyPattern": ["_id": 1],
      "keyValue": ["_id":"1563949999859302401"],
      "errmsg": "E11000 duplicate key      error collection: twitterdb.tweets       index: id_str_1 dup key: { id_str:      \"1563949999859302401\" }"
}
